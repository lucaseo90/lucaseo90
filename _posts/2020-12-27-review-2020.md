# Common
주요 업무: 마이크로서비스 구조로 설계된 플랫폼 내에서 데이터 엔지니어링관련 업무를 주로 진행했다.  
보조 업무: 주요 업무에 대한 신규 기능이 없는 경우 파트 내에서 신규 기술 조사, 세미나 및 파트 내 남은 백로그를 처리했다.

# ~ Mar. 2020
Keyword
* Data Engieering
* Spring Boot, Quartz Scheduler, Apche Nifi

고객 요구사항을 수립하여, 플랫폼 내의 데이터 이관 모듈을 담당해서 개발, 검증 및 운영(약 3개월)하다가 업무를 인수인계하고 복귀했다. 약 8개월간 파견 상태로 업무를 진행했는데 나름 장, 단점이 있었다. 간단하게 하나씩 정리하면 장점은 고객과 밀접하게 일하면서 개발한 소프트웨어가 실사용되는 것을 보니 보람을 느낄 수 있었던 것 같다. 반대로 일정이 생각보다 촉박하고, 출퇴근 시간 및 업무시간 증가로(통근 버스가 있긴 했지만..) 몸도 많이 상하고 육체적으로 많이 피곤했었다. 

# ~ Jun. 2020
Keyword
* Data Engieering
* Spring Boot, MSA to Monoltic

본사에 복귀하고 조직개편으로 새로운 파트에 속하게 되어 새로운 사람들이랑 업무를 진행했는데 참여하던 프로젝트 자체가 변경된 건 아니라서 기존에 맡던 데이터 이관 모듈을 담당해서 개발을 진행했다. 새로운 데이터 이관 엔진 추가해야 하는 요구사항과 주요 업무 외 업무로 마이크로서비스로 모듈이 나누어져 있었는데 세분화된 부분을 하나의 모듈로 통합해야 하는 요구사항이 있어서 해당 부분을 진행했다. 파견 직후 심신이 지친 상태였고, 개인 신변에도 변화가 생겨서, 능동적 보다는 수동적으로 생활하면서 출퇴근에 적응하던 시기였다.

개발자로 일한 지 5년이 지났고 6년 차에 접어들면서, 개발자로서 어떤 부분에 특별함을 가질지에 대한 생각을 많이 하는 시기가 찾아왔다. 

# ~ Sep. 2020
Keyword
* Backend Service
* Apache Flink, Apache Kafka

데이터 관련 모듈 기능 개발이 밀리면서 파트 내의 스트림 프로세싱(Stream Processing) 모듈 업무를 담당하게 되었다. 기능 개발이 필요했는데 확인해보니까 기존 기능에 버그가 있어서 버그도 수정하고, 기능 개발도 했다. 개념에 대해 이해하는 건 둘째치고, 어떻게 활용하는건지 파악하는데 시행착오가 많았었고, 분산처리 시스템이었기 때문에 단일 머신에서 실행할 때랑 클러스터 환경으로 구성했을 때 난해한 부분이 있었다. 결과적으로 업무로 진행해야 하는 기능은 잘 마무리 했다. 

개인 공부를 시작했는데  풀스택에 꽂혀서 JHipster를 공부했다. 풀스택을 지향한 이유는 백오피스 쪽이라도 간단하게 프론트 개발을 하고 싶다는 의향이었는데, JHipster 보면서 전반적인 풀스택에 대해서 개념이나마 잡을 수 있는 시간을 보냈던 것 같다. 

오픈소스 기여를 해보려고 DBeaver에 있는 이슈를 살펴보고, 버그를 해결해보려고 코드를 분석하고, 버그 픽스를 해서 PR까지 올렸는데, 반려됐다. 코드 분석하면서 전반적으로 잘 짜인(?) 코드를 보며 완벽하게 이해하지는 못했지만 잘 만들었다고 생각했고, 언젠가는 오픈소스에 commit 한 줄 남길 수 있다면 좋겠다고 생각했다. 그러면서 기존에 있는 코드의 기능적인 흐름까진 파악 하는 것 같은데, 작성자가 코드에 내포하고 있는 사상까지는 아직 이해하는데 부족하다는 성찰을 했다.

# ~ Dec. 2020
Keyword
* Survey & 
* Kubeflow, gRPC

크게 Kubeflow와 gRPC에 대한 업무를 진행하게 되었는데, 전반적인 개념, 설치 그리고 활용 정도에 대해 조사, 실습 및 문서화하는 작업을 주로 진행했다. 이와 관련해서 사내 세미나도 진행하면서 내가 이해하고 있는 것을 타인에게 설명하는 데 있어서 좀 더 구조적이고 논리적으로 이해시킬 수 있는 커뮤니케이션 스킬이 부족하다는 걸 느낄 수 있었다. 대학원 때 교수님이 `해당 기술에 배경지식이 없는 사람도 이해할 수 있게 글을 써야 한다.`라는 맥락으로 이야기하신 것이 생각났고, 회사와 관련해서 깊게 기술적인 연관이 없는 주제에 대해 기술적인 글을 블로그에 잘 작성할 수 있도록 해보려고 한다. 어떻게 하면 이런 부분을 채울 수 있을지는 더 생각해봐야겠다. 

출퇴근에 버스에서 보내는 시간이 아까워서, 출퇴근 중에는 주로 React를 공부하고 있다. 대략적인 개념은 파악해가고 있고, 무엇이든지 만들어봐야 더 깊게 이해할 수 있을 것 같은데 아직까진 무엇을 만들어봐야 할지 와닿지는 않고 있다.

데이터분석 준전문가(ADSP, **A**dvanced **D**ata Analytics **S**emi-**P**rofessional) 필기에 합격했다.

# Overall
올해는 그간 회사 일에 매달리면서 생활했다면, 자신에 대해서 더 많이 돌아보고 내 일상을 돌보려고 한 부분이 많은 것 같다. 그리고 한 해를 보내면서 많은 생각이 들었었는데, 당장 생각나는 건 `자기가 선택한 행동에 스트레스받을 거면 하지 말자`인 것 같다. 그리고 당장 내년 상반기까지는 세세한 계획보다는 그냥 순간순간 내 마음 가는 대로 생활해보려고 한다.
